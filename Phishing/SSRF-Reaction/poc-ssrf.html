[SSRF] Zero-Trust Browser Reaction Test

Tests browser-side SSRF protections — attempts to make the browser fetch internal/restricted resources via various techniques. A zero-trust browser should block requests to internal IPs, metadata endpoints, and dangerous URL schemes.

Test 1: Internal IP / Localhost Access

Tests if the browser blocks fetch/XHR requests to internal network addresses.

Test 2: Cloud Metadata Endpoint Access

Tests access to cloud provider metadata endpoints (AWS, GCP, Azure).

Test 3: URL Scheme / Protocol Abuse

Tests dangerous URL schemes that could be used for SSRF.

Test 4: IP Encoding / Obfuscation Bypass

Tests alternative IP representations that may bypass allowlist/blocklist filters.

Test 5: Redirect-Based SSRF

Tests if the browser follows redirects from external URLs to internal addresses.

Test 6: Resource Embedding (IMG, IFRAME, LINK, OBJECT)

Tests if HTML elements can load internal resources.

Test 7: DNS Rebinding Simulation

DNS rebinding attacks resolve to a public IP first (passing checks), then to an internal IP. The browser should detect this.
